## Notes for the design project (agent-utility framing)

The goal is for this page to feel like the agent's **own personal MLS** — every fact a buyer
or co-agent would ask for, on one screen, faster than logging into Stellar. Suggestions:

1. **Hero stat strip** — Price · Beds · Baths · Heated sqft · Lot · Year built, big and
   scannable, directly under the address. These are the six numbers agents quote on the phone.
2. **"At a glance" / spec table** — the grouped tables above (Location, Financial, Interior,
   Systems, Exterior) as collapsible or two-column sections. Keep MLS field names recognizable.
3. **MLS # + data-source line** — display `MLS# A4696731 · Stellar MLS` so it reads as authoritative.
4. **Updates timeline** — the dated improvements list is a strong differentiator; render it as a
   compact timeline. It answers the "what's been done to it?" question agents always field.
5. **One-tap actions for the agent** — Virtual tour link, "Copy MLS facts" button, and the photo
   tour already on the page. A print/PDF spec-sheet view would make this genuinely MLS-like.
6. **Keep marketing remarks separate from the data grid** — narrative reads as copy; the table
   reads as data. Agents trust the grid; buyers fall for the narrative.

**Do not publish as-is without owner sign-off on:** the listing office attribution (Preferred
Shore is the *listing brokerage*, not LF Property Media — make sure displaying their name/phone
is intended), and whether area-average comp figures should appear publicly.
